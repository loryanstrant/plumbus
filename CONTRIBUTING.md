# Contributing to PLUMBUS

Everyone knows about Plumbus, but contributions are always welcome! Here's how you can help make PLUMBUS even better.

## How to Contribute

1. **Fork the Repository**
   - Click the "Fork" button on GitHub
   - Clone your fork locally

2. **Create a Branch**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **Make Your Changes**
   - Follow the existing code style
   - Add humorous Rick & Morty references where appropriate (but keep it functional!)
   - Test your changes locally

4. **Test Your Changes**
   ```bash
   # Install dependencies
   pip install -r requirements.txt
   
   # Run the application
   python app.py
   
   # Build Docker image
   docker build -t plumbus-test .
   ```

5. **Commit and Push**
   ```bash
   git add .
   git commit -m "Add feature: description"
   git push origin feature/your-feature-name
   ```

6. **Create a Pull Request**
   - Go to GitHub and create a PR from your branch
   - Describe what you've changed and why
   - Reference any related issues

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable names (even if they reference Rick & Morty)
- Add comments for complex logic
- Keep the humor light and fun

## Reporting Bugs

If your Plumbus isn't working correctly:

1. Check the logs for error messages
2. Verify your SSH connectivity to clients
3. Open an issue on GitHub with:
   - Description of the problem
   - Steps to reproduce
   - Expected vs actual behavior
   - System information (OS, architecture, etc.)

## Suggesting Features

Have an idea to make PLUMBUS better? Open an issue with the "enhancement" label and describe:
- What feature you'd like to see
- Why it would be useful
- How it might work

## Questions?

Open an issue with the "question" label, and we'll help smooth out your dinglebop!
